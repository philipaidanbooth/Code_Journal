Journal Page About:

If you've reached this website, you're likely aware that I've both completed a marathon and shared insights on my training journey. This introduction outlines the framework behind each journal entry and the approach I plan to use for presenting them. Initially, let's delve into the journals' purpose. Keeping a coding journal is a well-recognized practice in computer science. Reflecting on my first CS class in high school, we maintained a daily code journal to foster personal growth and skill development. My goal with these journals is to extend the concept of a CS journal. See at my core, I'm a Classics major, which means the humanities deeply influence my perspective. Therefore, journaling will also be a way to chronicle my experiences and weave a narrative. I aim to selectively share journal entries each day, offering a glimpse into my journey in a manner that's accessible and engaging for anyone interested. My interdisciplinary approach has always enriched my understanding, serving as the driving force behind these entries.

Regarding the structure of each journal entry, my intention is to compose daily. However, I must acknowledge the potential for missed days. I've set a guideline for myself: not to skip more than two consecutive days. I imagine each entry to be approximately a page in length, double-spaced, covering topics such as daily training, academic reflections, and interpersonal relationships—though names will be kept confidential. All this is preliminary, I hope to flesh out a better structure after writing more of these entries, but yeah, that is pretty much all the pre-reading knowledge you’ve got to know before diving in.

For the end of today’s entry, I’m going to give a shot at explaining a circular list. Okay. When adding an element to an ArrayList in java, you put it at the index size of that list. But say you have an arraylist that has values in it and you’d like to insert an element into the beginning of that list. You would have to copy every element into a new array whose indices are shifted over one. This will require a for loop and will wound up with a big O notation of O(n). In layman's terms, it's not efficient. A circular arraylist aims to shrink the runtime of a method like addfirst from O(n) → O(1). Rather than intuitively holding the first index at the beginning of the list, the first index is recalculated every time an element is inserted into the front. It is calculated by decrementing the first index by one and then modulo (%) by the capacity. Modulo gives the remainder, so the distance from the previous first index can always be found, and since the method doesn’t use a for loop, the run time is O(1).
